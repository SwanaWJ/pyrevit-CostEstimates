# ----------------------------------------------------
# Commit Message Template (Conventional Commits)
# ----------------------------------------------------
# Format:
# <type>(<scope>): <short description>
# 
# Types:
#   feat     – a new feature
#   fix      – a bug fix
#   docs     – documentation only changes
#   style    – formatting, no code change
#   refactor – code change (not feature/bug)
#   test     – add/update tests
#   chore    – build process, tools, configs
#
# Scope: optional, e.g. (boq), (export), (readme)
# Short description: concise, imperative, no period
#
# Example:
# feat(boq): add Excel formula in Amount column
#
# ----------------------------------------------------
